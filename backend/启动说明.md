## 🧪 第3步：测试API

### 方法1：使用浏览器（推荐）

1. 打开浏览器访问：**http://localhost:8000/docs**
2. 你会看到Swagger API文档界面
3. 测试API：

#### 测试1：发送验证码
- 点击 `POST /api/v1/auth/send-code`
- 点击 "Try it out"
- 输入你的邮箱（如：`test@qq.com`）
- 点击 "Execute"
- 应该返回 200，并收到邮件

#### 测试2：注册用户
- 点击 `POST /api/v1/auth/register`
- 输入：
  ```json
  {
    "email": "test@qq.com",
    "code": "从邮件中获取的验证码",
    "password": "12345678",
    "nickname": "测试用户"
  }
  ```
- 点击 "Execute"
- 应该返回Token和用户信息

#### 测试3：登录
- 点击 `POST /api/v1/auth/login`
- 输入：
  ```json
  {
    "email": "test@qq.com",
    "password": "12345678"
  }
  ```
- 应该返回Token

### 方法2：使用PowerShell测试


## ❓ 可能遇到的问题

### Q1: 启动报错 "No module named 'app'"

**解决**：确保在backend目录下启动，且虚拟环境已激活

### Q2: 邮件发送失败

**可能原因**：
1. QQ邮箱密码不对（应该是授权码，不是登录密码）
2. 网络问题
3. QQ邮箱未开启SMTP服务

**验证授权码**：
1. 登录 https://mail.qq.com/
2. 设置 → 账户 → POP3/IMAP/SMTP服务
3. 开启服务并生成授权码

### Q3: 数据库连接失败

**检查**：
1. MySQL是否运行
2. 数据库tumor_screening是否存在
3. .env中的密码是否正确
